# Graphical Abstract Guide

## Overview

A graphical abstract is a single-panel visual summary of the paper. It appears in journal table-of-contents pages and social media shares. Many journals now require or encourage them.

---

## When Required

| Publisher / Journal | Status |
|--------------------|--------|
| Elsevier (most journals) | Required or encouraged |
| The Lancet family | Encouraged |
| JAMA Network | Not required |
| BMJ family | Not required |
| JMIR | Not required |
| Nature portfolio | Encouraged (as "summary figure") |
| Cell Press | Required |

Always check your target journal's author guidelines.

---

## Design Principles

1. One clear take-home message
2. Read left-to-right or top-to-bottom
3. Minimal text (max 30 words)
4. Large, simple icons and arrows
5. Consistent color scheme (2-3 colors)
6. No raw data tables or dense charts

---

## Common Layouts

### Layout 1: Problem to Solution (Most Common)

```
[Problem/Question] --> [Method] --> [Key Finding]
```

### Layout 2: Before and After

```
[Before / Without intervention]  vs.  [After / With intervention]
```

### Layout 3: Pipeline

```
[Input] --> [Process 1] --> [Process 2] --> [Output/Result]
```

### Layout 4: Central Finding

```
        [Context]
           |
[Method] - [KEY FINDING] - [Implication]
           |
        [Population]
```

## Technical Specifications

| Specification | Typical Requirement |
|--------------|-------------------|
| Size | 530 x 1200 pixels (Elsevier) or varies by journal |
| Resolution | 300 DPI minimum |
| Format | TIFF, PNG, JPEG, or PDF |
| Color | RGB for online |
| Font | Arial or Helvetica, 12+ pt |
| File size | Under 5 MB |

Always check your target journal for exact specifications.

---

## Content Checklist

- [ ] Single take-home message is clear
- [ ] No jargon or unexplained abbreviations
- [ ] Visual flow is intuitive (left-to-right or top-to-bottom)
- [ ] Consistent with the paper's conclusions
- [ ] Not a copy of any figure in the paper (should be unique)
- [ ] Patient/participant images have consent for publication
- [ ] Meets journal's size and format requirements
- [ ] Accessible to colorblind readers
